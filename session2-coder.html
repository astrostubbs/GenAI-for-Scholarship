Exercise 2: Local Jupyter Notebook with AI Integration

While Colab is convenient, many researchers prefer running Jupyter notebooks locally on their laptops for better control, offline access, and integration with local files. You can set up an AI chat assistant (Jupyternaut) directly inside JupyterLab, powered by Claude through Harvard's HUIT Bedrock proxy.

Jupyter AI Setup Guide (HUIT Bedrock)
Step-by-step instructions for configuring Jupyter AI with Harvard's API endpoint

Get your API key from the link at the top of this page.

Quick summary of what's needed:

  • Python 3.13+ installed on your laptop (download from python.org)
  • Virtual environment — python3 -m venv ~/jupyter-ai-env (keeps beta packages isolated)
  • Harvard HUIT API key — use the link above to retrieve your key
  • Jupyter AI v3 (beta) — pip install "jupyter-ai==3.0.0b9" boto3
  • Environment variable — AWS_BEARER_TOKEN_BEDROCK set to your API key

The full setup guide covers installation, environment configuration, Jupyternaut settings, available models, and troubleshooting.

Using AI in your local notebook: 

import anthropic
import os

# Initialize the client with Harvard endpoint
client = anthropic.AnthropicBedrock(
    aws_region="us-east-1"  # Not actually used, but required parameter
)

# Ask for help with your code
def get_ai_help(prompt):
    message = client.messages.create(
        model="us.anthropic.claude-opus-4-5-20251101-v1:0",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )
    return message.content[0].text

# Example: Ask AI to help debug
code_with_error = """
def calculate_mean(data):
    return sum(data) / length(data)  # Error: should be len()
"""

help_text = get_ai_help(f"Fix this Python code:\n{code_with_error}")
print(help_text)

Advantages of local notebooks with AI:

  • Work offline (after initial API calls)
  • Access to local files and databases
  • Full control over Python environment and packages
  • Integration with version control (git)
  • Better privacy for sensitive data

Exercise 3: Running AI-Integrated Notebooks on Harvard RC Cluster

For computationally intensive work, you may want to run Jupyter notebooks on Harvard's Research Computing (RC) cluster, which provides powerful hardware and can integrate with AI tools.

What is the RC Cluster?

Harvard Research Computing provides a high-performance computing cluster with:

  • Powerful CPU and GPU resources
  • Large-scale data storage
  • Pre-installed scientific software
  • Jupyter notebook access through Open OnDemand

Prerequisites:

  • Harvard RC account (request at rc.fas.harvard.edu)
  • VPN connection if accessing from off-campus
  • Basic familiarity with Linux command line
  • Harvard HUIT API key — see API setup guide

Accessing Jupyter on RC:

  1. Log in to Open OnDemand:
    • Go to rcood.rc.fas.harvard.edu
    • Log in with your Harvard credentials
  2. Launch a Jupyter notebook session:
    • Click "Interactive Apps" → "Jupyter Notebook"
    • Select partition (e.g., "test" for quick jobs, "shared" for longer runs)
    • Request resources (CPUs, memory, time limit)
    • Launch and wait for session to start
  3. Install AI libraries in your RC environment: 
    # In a terminal on RC (or in notebook cell with !):
pip install --user anthropic
  4. Configure Harvard API endpoint: 
    # Add to your ~/.bashrc on RC cluster:
export ANTHROPIC_BEDROCK_BASE_URL=https://apis.huit.harvard.edu/ais-bedrock-llm/v2
export ANTHROPIC_API_KEY="your-harvard-api-key"

# Then reload:
source ~/.bashrc

    Important: Never commit your API key to code repositories. Keep it in environment variables or secure configuration files that are excluded from version control.

Example workflow: Large-scale data analysis with AI assistance 

# In your RC Jupyter notebook
import anthropic
import pandas as pd
import numpy as np
import os

# Load large dataset (stored on RC cluster)
data = pd.read_csv('/n/holyscratch01/your_lab/large_dataset.csv')

# Initialize client with Harvard endpoint
client = anthropic.AnthropicBedrock(
    aws_region="us-east-1"  # Required parameter
)

prompt = f"""
I have a dataset with {len(data)} rows and columns: {list(data.columns)}.
Write Python code to:
1. Calculate correlation matrix
2. Identify top 5 most correlated variable pairs
3. Create a heatmap visualization
"""

response = client.messages.create(
    model="us.anthropic.claude-opus-4-5-20251101-v1:0",
    max_tokens=2000,
    messages=[{"role": "user", "content": prompt}]
)

print(response.content[0].text)
# Then run the generated code

Advantages of RC cluster for AI-assisted research:

  • Computational power: Handle large datasets that won't fit on your laptop
  • Parallel processing: Run multiple analyses simultaneously
  • GPU access: For ML/DL workloads that need acceleration
  • Persistent storage: Keep large datasets and results on cluster
  • Long-running jobs: Leave analyses running without keeping laptop on

Looking Ahead: Session 3 — Claude Code CLI

Next week, we'll explore Claude Code, a powerful command-line AI assistant that goes beyond inline suggestions. Unlike Gemini in Colab, Claude Code is an autonomous agent that can:

  • Execute multi-step tasks: Plan, implement, test, and debug complete workflows
  • Read and write files: Navigate your codebase and make changes across multiple files
  • Run commands: Execute Python scripts, install packages, run tests
  • Analyze real data: Work with data files and generate complete analysis pipelines
  • Create documentation: Generate reports, plots, and notebooks from your analysis

Claude Code represents a more autonomous approach to AI-assisted programming — you describe what you want to accomplish, and it plans and executes the entire workflow.

No setup required before Session 3 — we'll walk through installation together during that session.
